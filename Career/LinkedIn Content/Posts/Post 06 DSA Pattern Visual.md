The Two Pointers pattern has 4 variations. Here's when to use each.

[ATTACH: Diagram showing 4 variations]

1. Opposite ends (sorted array) → pair sum, container with most water
2. Fast-slow (same direction) → remove duplicates, detect cycle
3. Fixed sliding window → max sum of size k
4. Variable sliding window → longest substring without repeating chars

The visual that changed how I recognise patterns:

If the problem asks for a PAIR in a sorted array → opposite ends
If it asks for a RANGE satisfying a condition → sliding window
If it involves a linked list → fast-slow

120 problems in. These 4 templates cover roughly 30% of array/string mediums.
