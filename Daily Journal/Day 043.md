## DSA Block (2.5 hrs)
- Problem 1: [[1143 Longest Common Subsequence]] — LeetCode #1143 — Pattern: 2D String DP
  - Hint: `dp[i][j]` represents the LCS of `s1[0..i]` and `s2[0..j]`. If chars match: `1 + dp[i-1][j-1]`. If not: `max(dp[i-1][j], dp[i][j-1])`.
  - Complexity: Time O(m×n) | Space O(m×n)
- Problem 2: [[0072 Edit Distance]] — LeetCode #72 — Pattern: 2D String DP
  - Hint: Similar grid to LCS. If chars match, `dp[i][j] = dp[i-1][j-1]`. If they don't match, take the minimum of insert, delete, or replace operations + 1.
  - Complexity: Time O(m×n) | Space O(m×n)

## SQL Practice
- Topic: CASE WHEN and EXISTS vs IN vs JOIN performance
- Problem Statement: Categorize orders into 'High Value' (>1000) and 'Standard', and only for users who have an active subscription.
- Solution:
  ```postgresql
  SELECT o.id,
         CASE WHEN o.total > 1000 THEN 'High Value' ELSE 'Standard' END as category
  FROM orders o
  WHERE EXISTS (
      SELECT 1 FROM subscriptions s WHERE s.user_id = o.user_id AND s.status = 'ACTIVE'
  );
  ```
- Note: EXISTS is generally faster than IN for subqueries because it short-circuits on the first match, which is a common follow-up question.
