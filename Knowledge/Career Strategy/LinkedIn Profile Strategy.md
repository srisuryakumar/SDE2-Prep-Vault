# LinkedIn Profile Strategy

## The Profile as a Product
Your profile has three audiences:
1. **The Recruiter (5-8 secs):** Scans headline + title + skills for keywords.
2. **The Hiring Manager (20-30 secs):** Scans About and Experience for evidence you can do the specific job.
3. **The Peer Engineer (1-2 mins):** Reads your content and comments to see if you're a good colleague.

## Headline Formula
`[Current role] @ [Company] | [Specific specialization] | [What you're building/learning]`
*Example:* `Backend SDE @ [Company] | Building distributed payment systems with Java, Spring Boot, Kafka`
**Never use:** "Open to Opportunities" or "Looking for roles." It wastes precious keyword space and signals desperation.

## About Section Structure
1. **Hook:** A specific, concrete opening. (e.g. "I once spent a weekend tracing a 2-second latency spike...")
2. **Journey:** Path from where you started to what you build now.
3. **Value:** Specific technologies, scale, problem spaces.
4. **CTA:** What you are focused on and open to discussing.

## Experience Section
Unlike a resume, use this space to add context:
- Explain what the team owned and the problem space.
- Link to blogs, GitHub, or talks.
- Add texture to results (e.g., "...which mattered because this was the critical checkout path").

## Open to Work
Use **Recruiter-only** mode while employed. Set target titles, locations, and work-mode specifically to avoid spam. Avoid the green public banner, which signals desperation.

## Creator Mode & Algorithm
- Turn on Creator Mode and add 3 topic tags (#BackendEngineering, #SystemDesign).
- The algorithm rewards **engagement velocity in the first hour** and **consistency**.
- Native text/image content outperforms external links.

## Profile Audit Checklist (Key Items)
- Professional photo & custom banner.
- Headline contains keywords (e.g., Java, Spring Boot, Distributed Systems).
- Skills section has Java (or primary tech) pinned as #1.
- Featured section pins your GitHub or best technical post.
